Deploying your own personal cloud

Infinit differs from most other file systems and file storage services because of its decentralized nature. The following categorizes the file systems and storage services.

Centralized File Systems

Native file systems have been developed for years. At the time, such file systems were tightly coupled to the operating system they were developed for.




Centralized file systems operate by storing the data on the local hard disk, tape or whatever storage medium is provided, without relying on the network capabilities of the operating system.

These file systems suffer from the unreliable nature of storing data on a single non-fault-tolerant storage medium. However, such file systems are extremely fast since accessing data that is stored locally.

Distributed File Systems

The vast majority of cloud storage services work the same way as distributed file systems, which is by storing data on one or more servers. Clients request the data or files from those servers, making sure to maintain consistency, process popularly known as synchronization.




Noteworthy is that most cloud storage services do not provide a native file system interface. Also, cloud storage services differ from more traditional distributed file systems in the way they cache data. Indeed, while a distributed file system will cache only the most recently accessed data blocks, a cloud storage service often keeps all the files in order to provide better performance, leading to a large amount of local storage capacity being consumed.

Decentralized File Systems

Other solutions were developed in opposition to cloud storage services and distributed file systems in order to overcome the limitations in terms of reliability, scalability and fault tolerance.




By relying on the existing storage capacity left unused on most computers, decentralized file systems distribute the data blocks across the network rather than storing them all in a single place i.e a server/cloud.
