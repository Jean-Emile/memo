Overview

The Infinit Storage Platform allows for the creation of scalable, resilient and secure storage infrastructure for modern and container environments. This software-defined storage platform beneficing from several advantages over the centralized and distributed systems commonly used.

A resilient and self-healing architecture.

Decentralized

Unlike most distributed systems with a master/slave model, Infinit benefits from a truly peer-to-peer architecture protecting from single points of failure or bottlenecks.

Reliable and fault tolerant

Every block of every file is dynamically replicated several times (configurable by the administrator) in order to ensure durability and availability of the data.

In case of failures, Infinit will automatically adapt the number of replicas in order to guarantee the reliability of the data.

Scalable to peta bytes.

Manage your needs in terms of storage capacity by increasing/decreasing (scale up/down) contributions from existing providers and/or adding/removing (scale out/in) storage resources without interrupting the service.

Hyperconvergent vs hyperscalable

Scale your storage along with your application cluster to create a hyperconvergent architecture or keep a hyperscalable infrastrcture by dissociating your storage ressources from the rest of your cluster.

Programmatic.

Redundancy

Define the replication factor for every block of every file system object (file, directory etc.) to be stored multiple times.

Encryption

All the files stored are cut into chunk and every chunk is encrypted (RSA/AES-256) with a unique key.

Compression

Allow compression to be activated to reduce storage consumption depending on the nature of the data to be stored.

Data Placement

Control the placement of the redundant pieces of data through data placement policies: rack-aware, region-aware etc.

Deduplication

Provide a policy to activate deduplication in order to reduce storage consumption when storing identical pieces of data.

Versioning

Browse through the different versions of a file and rollback if necessary.

Adapt to your needs.

File System

Builds a low-level virtual disk interface on top of the DHT in order to provide a hierarchical organization of files, access-control, versioning for storing your entreprise documents, logs, and more.

Object Storage

Use an S3, GCS or OpenStack compatible object store to store your Wordpress media files.

Block Device

Deploy block device backends for your MongoDB database.
