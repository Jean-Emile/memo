A Resilient Architecture

Decentralized

Unlike most distributed systems relying on a master/slave model, Infinit benefits from a truly decentralized architecture (peer to peer). This model protects Infinit from single points of failure or bottlenecks.

Fault tolerant and self-healing

Every block of every file is dynamically replicated several times (configurable by the administrator) in order to ensure durability and availability.

Should a failure occur, Infinit will automatically adapt the number of redundant copies to come back to an healthy state.

Scalable To Petabytes

Manage your needs in terms of storage capacity by increasing/decreasing (scale up/down) contributions from existing nodes and/or adding/removing (scale out/in) storage servers without interrupting the service.

Programmable For Your Applications

Easily activate and configure policies for your needs.

Redundancy

Choose between a replication system by giving a replication factor, erasure coding or none.

Encryption

If activated, all the data items stored are encrypted (RSA/AES-256) and an access control mechanism is provided to manage keys.

Compression

Allow compression to be activated to reduce storage consumption depending on the nature of the data to be stored.

Data Placement

Control the placement of the redundant pieces of data through several policies: rack-aware, region-aware (different data centers) etc.

Deduplication

Provide a policy to activate deduplication in order to avoid storing identical pieces of data.

Versioning

Browse through the different versions of a file and rollback if necessary.

Adapt to your needs.

Block Device

Deploy low level virtual block device interfaces that can be formatted in the file system of your choice (ext4, NTFS, etc.) and used for non-competition flows such as databases.

File System

Access an hierarchocal organization of files through POSIX-compliant file system interfaces that provide advanced functionalities such as access control, versioning, etc.

Object Storage

Use an object storage interface (Amazon S3, OpenStack Swift) to access and store unstrctured data items aka blobs
