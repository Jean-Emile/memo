Get Started

The following is written for users of Mac OS X. If you are not using Mac OS X, please refer to the Linux and Windows guides.

Before proceeding, make sure you understand that this guide uses Infinit command­line tools with a terminal window. You don’t need to be a UNIX guru but you should be familiar with typing shell commands and interpreting outputs.

Download and install Infinit depedencies

Infinit relies on FUSE to create filesystems in userland. If not already, you will need install the OSX FUSE module by following the instructions:

Download OS X Fuse

Download and install the Infinit command-line tools

Please follow the link below to download the Infinit command ­line tools:

Download


Next, open your terminal and extract the Infinit tarball:

$> tar xjvf infinit­0.2.0.tbz
infinit­0.2.0/
infinit­0.2.0/bin/
infinit­0.2.0/lib
...
infinit­0.2.0/bin/infinit­storage
infinit­0.2.0/bin/infinit­user
infinit­0.2.0/bin/infinit­volume
$>

All the configuration files the Infinit command­line tools create and use are located in the $INFINIT_HOME directory which, by default, is set to $HOME/.infinit/. You can change edit your shell configuration to set INFINIT_HOME to another location if you want.

Note: We advise you to do so if you have the Infinit file transfer application on your computer as both applications would store their configuration files in $HOME/.infinit/.

Now that you’ve extracted the tarball, take a look. The extract directory contains subdirectories. bin/ contains the actual Infinit binaries such as infinit-user, infinit-network and more. The lib/ subdirectory contains all the libraries those binaries require to operate. Finally, the test/ subdirectory is provided for you to quickly test those command-line tools, see below.

$> cd infinit-0.2.0/
$> ls
bin/  lib/  test/
$>

Test by mounting an existing volume

As a first test, you are going to mount an existing volume that accesses data blocks located in a network we created ourselves. Don’t worry, this volume only contains test files and you cannot do any harm to those files since the test users only has read-only access.

Add the bin/ to the PATH environment variable to be able to invoke the command-line tools from anywhere:

$> export PATH=$PWD/bin/:$PATH
$>

Now run the infinit-volume command by prefixing it with the environment variable INFINIT_HOME=$PWD/../test/home to tell the command where to look for the configuration files required for this test.

$> INFINIT_HOME=$PWD/test/home/ infinit-volume --mount --name test --mountpoint $PWD/test/mountpoint/ -as alice
… XXX ...
$>

This command mounts the volume named test on behalf of the user ‘alice’ and makes it accessible from the mount point test/mountpoint/.

At this point, you can access the files in the test volume by browsing the test/mountpoint/ directory.

$> cd ../test/
$> ls
home/ mountpoint/
$> cd mountpoint/
$> ls
… XXX …
$>

Noteworthy is that test test volume contains 2GB of data. However, unlike cloud storage services like Dropbox, you were able to access its content without having to wait hours for all the files to be downloaded locally.

Now you can use both the terminal and your desktop graphical environment to access the files in the mount point. Try it!
