Introduction where you explain what is the purpose of this section talking about the reference of Infinit file system code base.

Nomenclature

The Infinit command-line tools are composed of several binaries, each dealing with a specific resource.

A user is represented through his/her RSA key pair. The user can create a network that represents the interconnection of one or more computers, servers and cloud services. A storage is a storage resource, local or remote, that can be connected to a network to support part of the overall storage load. Finally, a network can welcome several volumes i.e logical drives.

The hub is a cloud service whose role is to ease the process of discovery, sharing and more.

Home

All the configuration files Infinit creates and uses are located in the $INFINIT_HOME directory which, by default, is set to $HOME/.infinit/.

One can very easily set the environment variable to point to the directory of your choice, either by modifying your shell configuration or by setting it manually:

$> export INFINIT_HOME=”/some/where/”
$>

Terminology

All the command-line tools rely on the same terminology when it comes to options and arguments. It should therefore be pretty straightforward to start using a new binary. For instance, all the binaries have --create, --list, --fetch etc. options.

For more information on the options provided by a binary, do not hesitate to rely on the --help option. Noteworthy is that the help is relative to the options already provided. Therefore, do not hesitate to use the --help option with other options.

As an example, the help for infinit-user displays the general options available: export, create, import, register etc.:

$> infinit-user --help
Usage: infinit-user MODE [OPTIONS...]

Infinit user utility:

Modes:
  --export               Export a user for someone else to import
  --create               Create a user
  --import               Import a user
  --register             Create and register a user

Miscellaneous:
  -h [ --help ]          display the help
  -s [ --script ]        silence all extraneous human friendly messages
  -v [ --version ]       display version
$>

However, the help when invoking the --create option gives another output:

$> infinit-user --create --help
Usage: infinit-user --create [OPTIONS...]

Create a user:

Create options:
  -n [ --name ] arg      user name (defaults to system username)
  -k [ --key ] arg       RSA key pair in PEM format - e.g. your SSH key
                         (generated if unspecified)
$>

Hub

All entities created such as users, storages, networks, volumes etc. are created locally by default with no server involved. The creation process may generate one or more files and store them in the $INFINIT_HOME directory.

Only subsequently can a user decide to publish those entities to the hub. The Infinit hub is useful for users to easily retrieve their resources (credentials, networks, volumes etc.) on other devices.

Also, through the hub, users will be able to invite other users through their name rather than by manipulating user files manually. Note that in the future, other functionalities will be added to further simplify the process through the hub.

User

The infinit-user binary allows you to create, export and import users among other actions. A user is not much more than a RSA key pair that will be used to sign and encrypt data.

The following creates a user, generating a RSA key pair for you. Note that the name of the user is deduced from the system:

$> echo $USER
chiche
$> infinit-user --create
Generating RSA keypair.
Generated user "chiche".
$>

You can specify the name of the user through the --name option:

$> infinit-user --create --name chodaboy
Generating RSA keypair.
Generated user "chodaboy".
$>

Likewise, if you want Infinit to use your SSH RSA key rather than generating a new one:


$> infinit-user --create --name chodaboy --key ~/.ssh/id_rsa
Passphrase:
Generated user "chodaboy".
$>

WARNING: Keep in mind that the user-related data must never fall into someone else’s hands. The user file is the equivalent of an SSH key file and must therefore be kept private at all times.

You may want to export a user, to transfer it to another computer for instance. Even though Infinit provides a way to move resources around through the Infinit hub, such a functionality is not provided for users given their sensitive nature.
